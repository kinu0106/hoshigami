# はじめに読む：このプロジェクトのVPS環境について

このドキュメントでは、VPS環境の全体像と、各ドキュメントの読む順番を説明します。
初めてこのプロジェクトのVPS作業を行う方は、まずこの文書を読んでから作業を開始してください。

---

## 📌 このプロジェクトについて

本プロジェクト（`hoshigami`）は、エックスサーバーVPS上で動作しています。  
1つのドメイン（`https://furihaba.net`）の `/hoshigami/` 以下に、複数のシステムを並べて運用しています。

---

## 🗺️ サーバー上のディレクトリ構造（全体像）

```text
/var/www/
  │
  ├── html/                    ← デフォルトのWeb画面（基本的には触らない）
  │
  ├── hoshigami_portal/        ← 🔐 認証ポータル画面（/hoshigami/ に表示される）
  │
  ├── hoshigami/               ← 📊 実績システム（現在稼働中）
  │   ├── frontend/            ← 画面側（Next.js）
  │   ├── backend/             ← API側（Express.js）
  │   └── ecosystem.config.js  ← プロセス管理の設定ファイル
  │
  └── hoshigami_〇〇/          ← 🚀 今後追加される新しいシステムの置き場所
      ├── frontend/
      ├── backend/
      └── ecosystem.config.js
```

## 🔄 開発〜本番反映の全体の流れ

```
手元のPC でコード修正
       ↓
  GitHub に Push
       ↓
  VPS にSSHで入る
       ↓
  git pull（最新コードを取得）
       ↓
  npm run build（フロントエンドの組み立て）
       ↓
  pm2 reload（サービスの再起動）
       ↓
 ブラウザで動作確認 ✅
```

> **【重要】** GitHubにPushしただけでは本番は更新されません。
> 必ずVPSにログインして更新作業を行う必要があります。

## ⚠️ 開発者が守るべき主なルール

新しいシステムを追加する際に守るべきルールは以下の通りです。詳細は各担当者に確認してください。

- **URLパス**: `/hoshigami/〇〇/` 以下に配置（サーバー管理者と相談して決定）
- **ポート番号**: 他システムと重複しないように割り当て（現状 `3000`, `5001` は使用中）
- **データベース**: システムごとに別のDBを使用（既存DBの使い回し禁止）
- **プロセス管理**: PM2 + `ecosystem.config.js` で管理
- **Nginx設定**: 変更はサーバー管理者に依頼

> 新システムを追加する前に、必ずサーバー管理者にポート番号・DB・ディレクトリの割り当てを確認してください。
