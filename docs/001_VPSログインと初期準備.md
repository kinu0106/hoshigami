# フルスクラッチ開発：VPSログインと初期準備（初心者向け）

このドキュメントでは、初めてVPS（本番サーバー）にアクセスする人に向けて、必要なソフトの準備と接続方法を説明します。
システムを本番環境（VPS）に切り替える「デプロイ」作業を行う前に、まずはサーバーの中身を「見れる」「操作できる」状態にする必要があります。

---

## 1. サーバーの中身を「目で見る」準備（WinSCP等）
本番サーバー上にあるファイル（画像やコード）を、Windowsのエクスプローラーのように一覧で確認したり、ファイルを直接編集したりするには「SFTPソフト」を使います。

*   **Windowsの方向け**: **WinSCP（ウィンエスシーピー）** が主流で使いやすくおすすめです。
*   **Macの方向け**: Cyberduck や FileZilla などが定番です。

ここでは、Windowsで **WinSCP** を使う場合の設定方法を解説します。

### WinSCPの接続設定
WinSCPを立ち上げると、最初に「ログイン」や「新しいサイト」という画面が出ます。
以下のように入力してサーバーに接続（ログイン）してください。

*   **転送プロトコル**: `SFTP` を選択します。
*   **ホスト名**: `furihaba.net` （VPSのドメイン名です）
*   **ポート番号**: `58923` （※デフォルトの22番ではなく、セキュリティのために変更されています。要注意！）
*   **ユーザ名**: `root` （一番偉い管理者権限のユーザーです）
*   **パスワード**: （サーバー管理者から共有されたパスワードを入力します）

> **【注意ポイント】**
> ポート番号の「22」のまま繋ごうとしても弾かれてしまいます。必ず「**58923**」に変更してください。

「保存」ボタンを押して「本番サーバー」などの名前をつけておくと、次回からパスワードなし（またはワンクリック）で接続できるようになります。

---

## 2. サーバーを「文字で操作する」ための準備（SSH接続）
WinSCPは「見る」ためのソフトですが、実際に「システムの最新版ダウンロード（`git pull`）」や「システムの再起動（`npm run build`）」などの複雑な命令は、ターミナルから文字を打ち込む必要があります。これを **SSH接続** と呼びます。

特別なソフトは不要で、皆さんが普段使っている **VS Code のターミナル**（PowerShellなど）からそのまま接続できます。

### SSH接続のコマンド
ターミナルを開き、以下のコマンドを打ち込んでEnterキーを押します。

```bash
ssh -p 58923 root@furihaba.net
```
パスワードを聞かれたら、先ほどWinSCPで入力したのと同じパスワードを入力します。（※文字を打っても画面には何も表示されませんが、ちゃんと入力されているのでそのままEnterを押してください）

接続に成功すると、左側の文字が `root@xxxx:~#` のような表記に変わり、ここから先は「あなたのPC」ではなく「本番サーバーの中」を操作している状態になります。

## 3. GitHubを使った更新の準備
**「GitHubを使ったデプロイ」**をルールとしています。

1. **手元のPC（VS Code）** ：“コードを書き換える” → “GitHub（共有の倉庫）へ送る（Push）”
2. **本番サーバー（VPS）** ：“GitHubから最新のコードを取り寄せる（Pull）” → “再起動する”

具体的なコマンドの手順（どうやってPushして、どうやってPullするか）については、
別のドキュメント **『[002_VPS初回デプロイ手順.md](./002_VPS初回デプロイ手順.md)』** を参照してください。
