# 新規プロジェクトの初回デプロイとGitHub連携

このドキュメントでは、まったく新しいシステム（`hoshigami_〇〇`など）を立ち上げて、**初めてVPSサーバーに配置するまでの手順**を解説します。

普段の更新は `git pull` コマンド一発で終わりますが、**「最初の一回目」**だけは、サーバーとGitHubに「お互いを信用してやり取りしていいよ」と教える設定（鍵の登録と初回ダウンロード）が必要です。

---

## 1. 概念の理解：なぜ連携が必要か？
普段、自分のPC（VS Code）でプログラムを書いてGitHubにアップロード（Push）できているのは、皆さんのPCに「GitHubにアクセスする権限（鍵）」が設定されているからです。

同じように、VPSの本番サーバーも「GitHubからコードをダウンロード（Pull）する権利」を持たなければなりません。
※本番サーバーの場合は、全員が見れる「公開（Public）リポジトリ」ではなく「非公開（Private）リポジトリ」からコードを取ってくるため、この連携が必須になります。

---

## 2. サーバー用の「鍵」を作成する
サーバーがGitHubのドアを開けるための「合鍵（SSHキー）」を作ります。
VS Codeのターミナルなどから**VPSにSSH接続した状態**（ターミナルで `root@xxxx:~#` と表示されている状態）で、以下のコマンドを実行します。

```bash
# 鍵を作るコマンド（エンターを3回押して進めます）
ssh-keygen -t ed25519 -C "VPS-Deploy-Key-〇〇"
```
（※ `-C` の後ろの `"VPS-Deploy-Key-〇〇"` の部分は、ただの「コメント（名前付け）」なので、システム名などに合わせて**自由に変更してOK**です。後でGitHubの画面を見た時に「あ、これがあのシステムの鍵だな」と分かりやすくするためのメモです）
（※途中でパスワードなどを求められますが、すべて何も入力せずに「Enterキー」でスキップしてOKです）
（※ この作業はVPSにログインした状態で、**サーバーの中に**鍵を作る作業です）

### 💡 コマンドの各パーツの意味
このコマンドの各要素の意味を解説します。

*   **`ssh-keygen`**: 鍵生成プログラムを実行するコマンド（**固定**）
*   **`-t ed25519`**: 生成する鍵の種類（アルゴリズム）の指定（**推奨設定**）
*   **`-C`**: コメント（メモ）を記述するためのオプション（**固定**）
*   **`"VPS-Deploy-Key-〇〇"`**: 鍵を識別するための任意の**コメント**（**自由に変更可能**）

#### 🛡️ なぜ `ed25519` なの？
以前は `RSA` が主流でしたが、現在は `ed25519` が推奨されています。
*   **安全性**: `RSA` よりも鍵長が短いながら、解読が非常に困難です。
*   **高効率**: データのサイズが小さいため、認証時の計算が高速です。
*   **利便性**: 文字列が短いため、コピー＆ペースト等の取り扱いが容易です。
（※古いサーバーだと対応していないこともありますが、今のVPSやGitHubなら100%これを使っておけば間違いありません）

（※コマンドを打った後、途中でパスワードなどを求められますが、すべて何も入力せずに「Enterキー」でスキップしてOKです）

出来上がった「公開鍵」の中身を表示します。
```bash
cat ~/.ssh/id_ed25519.pub
```
画面に `ssh-ed25519 AAAA...` のような長い文字列が表示されるので、それを全てコピーします。

---

## 3. GitHubに鍵を登録する（Deploy keys）
今度はブラウザで **GitHub** にアクセスして、作った合鍵を使えるように登録します。

1. 追加したいプロジェクトのリポジトリのページを開きます。
2. 上部の `Settings`（歯車マーク）をクリックします。
3. 左サイドバーから `Deploy keys` をクリックします。
4. 右上の `Add deploy key` ボタンを押します。
5. 以下を入力します：
   * **Title**: `VPS Deploy Key` など分かりやすい名前
   * **Key**: 先ほどコピーした長い文字列を貼り付けます
   * ※「Allow write access（書き込みも許可する）」のチェックは不要です。サーバーはダウンロード（Pull）するだけなので安全第一にします。
6. `Add key` を押して保存します。

これでお互いの準備（連携）は完了です！

---

## 4. サーバーに初めてコードを持ってくる（Clone）
いよいよ、サーバーにプロジェクトのフォルダを作って、GitHubからコードを丸ごとダウンロードします。これを **Clone（クローン）** と呼びます。

### ⚠️ 大事なポイント：この鍵は「このリポジトリ専用」！
今回 GitHub の `Settings -> Deploy keys` に登録した鍵は、**その特定のリポジトリ専用の合鍵**になります。

*   **1つのシステム（リポジトリ）＝ 1つのデプロイキー** が原則です。
*   別の新しいシステム（別のリポジトリ）を同じVPSにデプロイする場合は、本来はまた新しい鍵を作るか、管理者として「ユーザー全体の鍵（SSH Keys）」を使う設定が必要になります。
*   よく分からないうちは、「システムを新しく作るたびに、この連携作業を繰り返すんだな」と思っておけば間違いありません！



引き続きVPSにSSH接続したターミナルで、以下の順番にコマンドを打ち込みます。

```bash
# Web公開用のフォルダ（みんなの部屋が集まる場所）に移動
cd /var/www/

# GitHubからコードの箱を丸ごとダウンロードする（コピーしたURLを貼り付けます）
git clone git@github.com:〇〇/hoshigami_〇〇.git  # ここを自分のURLに書き換える

   💡 この自分のURLはどこで確認するの？
   GitHubの画面から、自分のプロジェクト専用の正しいURLをコピーできます。

   1. GitHubのリポジトリページを開きます。
   2. 緑色の `<> Code` ボタンをクリックします。
   3. **「SSH」**タブを選択します（※重要！HTTPSではなくSSHを選んでください）。
   4. `git@github.com:...` と書かれたURLをコピーします。

# ダウンロードしたフォルダの中に移動する
cd hoshigami_〇〇  # ここも自分のフォルダ名に書き換える
```

※最初に `Are you sure you want to continue connecting (yes/no/[fingerprint])?` と聞かれたら、落ち着いて `yes` と入力してEnterを押してください。

これで、VPS上に `/var/www/hoshigami_〇〇/` というフォルダが完成し、最新のコードがサーバー内に配置されました。

### 次にやること：環境変数の設定
ダウンロードした直後は、サーバー専用のパスワードなどが設定されていません。
引き続き、**『[003_VPS環境変数(.env)の設定.md](./003_VPS環境変数(.env)の設定.md)』** を見ながら、本番用の設定ファイルを作成してください。

そこまで終わったら、最後に『VPS日常のデプロイ手順書.md』などを参照してシステムを起動（Build/Reload）させます！
