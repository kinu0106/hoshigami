# VPS環境変数（.env）の設定

本番サーバー（VPS）でシステムを動かすためには、**「サーバー専用の設定（パスワードやURL）」**を書き込んだ `.env` というファイルを用意する必要があります。

このファイルはセキュリティ上 GitHub には送られないため、**サーバー上で手動で作成（または修正）**する必要があります。

---

## 💡 環境変数の「2つのルール」

環境変数の性質を整理します。

### 1. 「項目」は同じ、「値」が違う
基本的には、手元のPC（ローカル）の `.env` に書いてある**名前（PORTやDB_USERなど）**は、本番でも同じものを使用します。
ただし、右側の**中身（5002や本番パスワードなど）**だけを、本番環境に合わせて書き換える必要があります。

---

## 🔍 WinSCP で .env が見当たらない場合

`.env` のように、ファイル名の先頭に `.`（ドット）がつくファイルは、Linux では「隠しファイル」扱いになります。

### 解決方法
WinSCP の画面上で **`Ctrl + Alt + H`** を押してください。
隠れていたドットファイルが表示されるようになります。

---

### 1. 新しくファイルを作る
WinSCP で対象のフォルダ（`frontend` や `backend`）の中で右クリックし、**「新規」→「ファイル」** を選びます。
ファイル名は **`.env`** と入力してください。

### 2. 本番用の値を書き込む
作成したファイルをダブルクリックして開き、以下のテンプレートをコピー＆ペーストした上で、自身の本番環境に合わせて書き換えてください。

#### 【バックエンド（backend）用のテンプレート】
```text
# ⚠️ ポート番号は他のシステムと絶対にかぶらないように設定してください
# （例：実績システムが5001なら、新システムは5002にする等）
PORT=（割り当てられた番号を入力）

CORS_ORIGIN=https://furihaba.net
SESSION_SECRET=（適当な長い英数字を自分で決めて入力）

DB_HOST=localhost
DB_PORT=5432
DB_USER=（サーバー管理者に聞いた本番用ユーザー名）
DB_PASSWORD=（サーバー管理者に聞いた本番用パスワード）
DB_NAME=（サーバー管理者に聞いた本番用DB名）

など、プロジェクトの.envで設定した項目を本番用に書き換える
```

#### 【フロントエンド（frontend）用のテンプレート】
```text
# frontend側で必要な設定があればここに記述します
# 例：APIの宛先など
NEXT_PUBLIC_API_URL=https://furihaba.net/hoshigami/〇〇/api
```

### 3. 保存して閉じる
保存した瞬間、サーバー上のファイルが更新されます。

---

## 🎨 Next.js（フロントエンド）特有の注意点

Next.js プロジェクトでは、`.env.local` や `.env.production` など複数の環境変数ファイルが混在することがよくあります。

### どれが本番で使われるの？
Next.js には「読み込み優先順位」があります。
1.  **`.env.production`** : 本番モード（`npm start`）の時に**最優先**で読み込まれます。
2.  **`.env.local`** : 主に開発中の自分のPCで使用するものです。

### 結局どうすればいい？
初心者の人が迷わないための推奨設定は以下の通りです。
- **「本番サーバー」には `.env` という名前のファイルを1つだけ作成する**

`.env.production` があればそれを編集しても問題ありませんが、判断に迷う場合は `.env` 1本にまとめてしまえば、開発モードでも本番モードでも確実に読み込まれます。

---

## ⚠️ 注意事項

- **GitHub には送らない**: 
  自分の PC でこのファイルを編集して `git push` しようとしても、通常は無視される設定（`.gitignore`）になっています。
  **「本番の設定変更は、本番サーバーにログインして直接行う」**というルールを忘れないでください。
- **反映には再起動が必要**:
  `.env` を書き換えただけではシステムは古い設定のまま動いています。必ず **PM2 のリロード（再起動）** を行ってください。
  （詳細は『004_VPS日常のデプロイ手順書.md』を参照）
