# VPS日常のデプロイ手順書（プログラムの更新）

このドキュメントでは、手元のPCで書き換えたプログラムを、**GitHubを経由して本番サーバー（VPS）に反映させる日常の流れ**を解説します。

---

## 🏗️ 全体の流れ

1.  **ローカルPC**: コードを修正し、**GitHubへプッシュ（Push）**する
2.  **本番サーバー**: GitHubから**最新版を取得（Pull）**する
3.  **本番サーバー**: アプリケーションを**ビルド（Build）**する
4.  **本番サーバー**: サービスを**再起動（Reload）**する

> **【重要】自動適用はされません**
> GitHubにプッシュしただけでは本番環境は更新されません。必ずサーバーにログインして更新作業を行う必要があります。

---

## 【STEP 1】手元のPC（VS Code）での作業

修正が終わったら、以下のコマンドをVS Codeのターミナルで順番に打ちます。

```powershell
# 1. 変更したファイルを記録の対象にする（ステージング）
git add .

# 2. 何を直したかメモを添えて保存する（コミット）
git commit -m "修正内容のメモ（例：タイトルの色を変更）"

# 3. GitHubへ送り出す（プッシュ）
git push origin main
```

---

## 【STEP 2】本番サーバー（VPS）での作業

GitHubに送られた最新のコードを、本番環境に取り込みます。

### 1. サーバーに接続して、プロジェクトの部屋に移動する
VS CodeのターミナルからSSHでログインし、対象のフォルダへ移動します。

```bash
# サーバーにログイン（パスワードを入力）
ssh -p 58923 root@furihaba.net

# プロジェクトのディレクトリへ移動
cd /var/www/hoshigami_〇〇(今回作成するシステム用ディレクトリ)
```

### 2. GitHubから最新版をダウンロードする
```bash
git pull origin main
```

> **💡 もし環境変数の変更が必要な場合は？**
> 新しい機能の追加などで、本番環境のパスワードや設定（.env）を変える必要がある場合は、**組み立て（Build）の前に**作業を行います。
> 詳細は **『[003_VPS環境変数(.env)の設定.md](.003_VPS環境変数(.env)の設定.md)』** を参照してください。

### 3. フロントエンド（画面側）を組み立てる
画面（見た目）を修正した場合は、この「組み立て（ビルド）」作業が必須です。
※完了まで数十秒〜数分かかることがあります。

```bash
cd frontend
npm run build
cd ..
```

### 4. システムを再起動して反映させる
最後に、プロセス管理人（PM2）に最新の状態を読み込ませます。

```bash
# 指示書（ecosystem.config.js）を使って再起動
pm2 reload ecosystem.config.js

# 今の起動状態を保存（サーバーが再起動しても自動で立ち上がるようにする）
pm2 save
```
