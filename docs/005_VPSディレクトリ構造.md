# VPSディレクトリ構造（全体像）

本ドキュメントでは、エックスサーバーVPS上のプロジェクトの全体的なディレクトリ配置（`/var/www` 以下）を図解しています。
今後新しいシステムを追加する際に、ファイルがどこに配置されるのか、設定ファイルがどこにあるのかの参考にしてください。

---

## 📍 サーバー上の配置図

```text
/var/www/
  │
  ├── html/                  ← （デフォルトのWeb画面など。基本的には触らない場所）
  │
  ├── hoshigami_portal/      ← 🔐 認証ポータル画面用
  │     （ ブラウザで https://furihaba.net/hoshigami/ にアクセスした時に表示される画面の置き場所 ）
  │
  ├── hoshigami/             ← 📊 現在の実績システム
  │   │                         （※ 将来的にフォルダ名を hoshigami_results に変更する予定です）
  │   │
  │   ├── frontend/          ← 画面側（Next.js等）のコード
  │   ├── backend/           ← 裏側API（Node.js等）のコード
  │   └── ecosystem.config.js ← ★ 実績システム用のプロセス管理設定
  │                             （「hoshigami-backend」や「hoshigami-frontend」を起動する指示書）
  │
  │
  └── hoshigami_〇〇/        ← 🚀 （例：今後新しく作る「〇〇」システムの置き場所）
      │
      ├── frontend/          ← 新システムの画面側のコード
      ├── backend/           ← 新システムの裏側APIのコード
      └── ecosystem.config.js ← ★ 新システム専用のプロセス管理設定
                               （「hoshigami-〇〇-frontend」等を起動する指示書）
```
