# 星上通信実績集計システム 開発プラン

## プロジェクト概要

- **目的**: デジアイズへ出す請求書のデータ集計
- **構成**: フロントエンド（Next.js）とバックエンド（Express.js）の分離構成
- **デプロイ環境**: 社内サーバー
- **OS**: Ubuntu 24.04 LTS（予定）
- **データベース**: PostgreSQL（社内サーバー運用）
- **将来展開**: 在庫管理システムとの連携を考慮

## 技術スタック

### Backend

- Express.js + TypeScript
- pg（node-postgres） - PostgreSQLクライアント
- express-validator
- express-session (セッション管理)
- cors, dotenv

### Frontend

- Next.js 15 (App Router)
- React 19 + TypeScript
- Bootstrap CSS
- react-hook-form

## 社内サーバーの設定

### サーバー環境

- **OS**: Ubuntu 24.04 LTS（予定）
- **データベース**: PostgreSQL（社内サーバー上で運用）

## データベース設計（PostgreSQL）

### テーブル構成

1. **departments** (製造依頼元マスタ)
   - id SERIAL PRIMARY KEY
   - name VARCHAR(255) UNIQUE NOT NULL

2. **employees** (入力者マスタ)
   - id SERIAL PRIMARY KEY
   - name VARCHAR(255) UNIQUE NOT NULL

3. **products** (製品マスタ)
   - id SERIAL PRIMARY KEY
   - name VARCHAR(255) NOT NULL
   - unit_price NUMERIC(10, 2) NOT NULL
   - department_id INT NOT NULL REFERENCES departments(id)
   - UNIQUE (department_id, name)

4. **achievements** (実績データ)
   - id SERIAL PRIMARY KEY
   - input_date DATE NOT NULL
   - employee_id INT NOT NULL REFERENCES employees(id)
   - department_id INT NOT NULL REFERENCES departments(id)
   - product_id INT NOT NULL REFERENCES products(id)
   - quantity INT NOT NULL
   - created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
   - updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

## 画面構成

### 1. ログイン画面 (`/login`)

- パスワード入力（2008）
- セッション管理

### 2. ホーム画面 (`/home`)

- 「実績一覧」「実績入力」「実績修正」へのナビゲーション
- **認証が必要な画面**: 一覧画面（`/list`）、入力者管理画面（`/employees`）、製品管理画面（`/products`）
- **認証不要な画面**: 実績入力画面（`/input`）、実績修正画面（`/edit`）

### 3. 一覧画面 (`/list`) ※管理者のみ

- 日付 & 製造依頼元でフィルター（必須）
- 表示項目: 入力日、入力者、製造依頼元、製品名、台数、単価、集計
- 合計金額表示

### 4. 実績入力画面 (`/input`)

- 入力者選択（ドロップダウン）
- 当日日付表示
- 製造依頼元 → 製品名（連動）→ 台数
- 複数件まとめて入力可能
- 確認画面（`/input/review`）で保存前確認

### 5. 実績修正画面 (`/edit`)

- ページネーション付き一覧
- フィルター機能
- 編集画面への遷移

### 6. 入力者管理画面 (`/employees`) ※管理者のみ（ログイン必要）

- 入力者マスタの一覧表示
- 入力者の追加・更新・削除機能

### 7. 製品管理画面 (`/products`) ※管理者のみ（ログイン必要）

- 製品マスタの一覧表示
- 製品の追加・更新・有効/無効切り替え機能

## API設計

### エンドポイント

- `POST /api/auth/login` - ログイン
- `GET /api/departments` - 製造依頼元一覧
- `GET /api/employees` - 入力者一覧
- `GET /api/products` - 製品一覧（department_idでフィルター可）
- `GET /api/achievements` - 実績一覧（フィルター対応）
- `POST /api/achievements` - 実績登録（複数件対応）
- `PUT /api/achievements/:id` - 実績更新
- `DELETE /api/achievements/:id` - 実績削除
- `GET /api/summary` - 集計データ取得

### マスタ管理API（管理画面用）

- `POST /api/departments` - 製造依頼元追加
- `PUT /api/departments/:id` - 製造依頼元更新
- `POST /api/products` - 製品追加
- `PUT /api/products/:id` - 製品更新
- `POST /api/employees` - 入力者追加
- `PUT /api/employees/:id` - 入力者更新

#### 本番環境デプロイ時の注意事項

**請求書番号の初期化**
- 本番環境にデプロイする前に、`invoice_numbers`テーブルの`current_number`を`0`に設定してください
- これにより、最初の請求書番号が`000001`から始まります
- SQL: `UPDATE invoice_numbers SET current_number = 0;`
