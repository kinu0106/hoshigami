# ログイン機能の問題 - 状況まとめ

## 問題の概要
HTTPS環境（`https://furihaba.net`）でログイン機能が動作しない。
- ログインAPIは成功（200 OK、`{ok: true}`）
- バックエンドのログでは「Login successful, session set」と表示される
- しかし、レスポンスヘッダーに`Set-Cookie:`が含まれていない
- そのため、ブラウザにセッションクッキーが設定されず、ログイン後も認証状態が維持されない

## 原因の特定と修正

### 根本原因
1. **`saveUninitialized: false`の設定**
   - `express-session`では、`saveUninitialized: false`の場合、セッションが変更されていても、初期化されていないと判断される場合があり、クッキーが送信されない
   - 特に`rolling: true`と組み合わせた場合、この問題が発生しやすい

2. **セッション保存のタイミング**
   - セッションに値を設定した直後にレスポンスを送信していたため、セッション保存が完了する前にレスポンスが送信される可能性があった

3. **クッキー設定**
   - HTTPS環境では`secure: true`と`sameSite: 'none'`の組み合わせが必要（クロスオリジン対応）

### 実施した修正

#### 1. バックエンドのセッション設定（`backend/src/index.ts`）
- `saveUninitialized: false` → `saveUninitialized: true`に変更
- `secure: process.env.SESSION_SECURE === 'true' || false` → `secure: process.env.SESSION_SECURE === 'true'`に修正
- `sameSite: 'lax'` → `sameSite: process.env.SESSION_SECURE === 'true' ? 'none' : 'lax'`に変更（HTTPS環境では`'none'`が必要）

#### 2. 認証コントローラー（`backend/src/controllers/authController.ts`）
- セッション保存を明示的に待つ処理を追加
- `req.session.save()`コールバック内でレスポンスを送信するように変更
- これにより、`Set-Cookie`ヘッダーが確実に送信される

### Nginx設定

## 修正内容の詳細

### `backend/src/index.ts`
```typescript
session({
  // ...
  saveUninitialized: true, // trueに変更：セッションクッキーを確実に送信するため
  cookie: {
    secure: process.env.SESSION_SECURE === 'true', // HTTPS環境ではtrueである必要がある
    sameSite: process.env.SESSION_SECURE === 'true' ? 'none' : 'lax', // HTTPS環境では'none'が必要
  },
})
```

### `backend/src/controllers/authController.ts`
```typescript
export function login(req: Request, res: Response) {
  // ...
  req.session.isAdmin = true;
  
  // セッション保存を明示的に待ってからレスポンスを送信
  req.session.save((err) => {
    if (err) {
      console.error('Session save error:', err);
      return res.status(500).json({ ok: false, message: 'Session save failed' });
    }
    res.json({ ok: true });
  });
}
```
