# 星上通信実績集計システム 開発フロー

## 全体の流れ

### サーバー承認前（今すぐできる）
```
Phase 1: バックエンド基盤構築
├─ PostgreSQLローカルインストール（pgAdmin併用）
├─ Express.js + TypeScriptプロジェクト作成
├─ データベーススキーマ作成
└─ ダミーデータ投入

Phase 2: バックエンドAPI実装
├─ 認証API
├─ マスタ系API
└─ 実績系API

Phase 3: フロントエンド基盤構築
├─ Next.jsプロジェクト作成
└─ APIクライアント実装

Phase 4: フロントエンド画面実装
├─ ログイン画面
├─ ホーム画面
├─ 実績入力画面
├─ 実績修正画面
└─ 一覧画面（管理者用）

Phase 5: 統合テスト
└─ ローカル環境での動作確認
```

### サーバー承認後（社長承認が必要）
```
Phase 6: VPS環境構築 & デプロイ
├─ エックスサーバーVPS契約
├─ Ubuntu環境構築
├─ PostgreSQLインストール
├─ Node.js環境構築
├─ アプリケーションデプロイ
└─ 本番環境での動作確認
```

---

## 詳細フロー

### Phase 1: バックエンド基盤構築

#### Step 1-1: ローカル環境の準備
```
□ PostgreSQLをPCにインストール
□ データベース「hosh有以下」を作成
□ テーブル作成スクリプトを準備
```

#### Step 1หมู่プロジェクトの作成
```
□ backend/ディレクトリを作成
□ npm init -y
□ TypeScriptの設定
□ Express.jsのインストール
□ PostgreSQLクライアント（pg）のインストール
```

#### Step 1-3: データベーススキーマ作成
```
□ departments テーブル作成
□ employees テーブル作成
□ products テーブル作成
□ achievements テーブル作成
```

#### Step 1-4: ダミーデータ投入
```
□ 製造依頼元マスタのダミーデータ
□ 製品マスタのダミーデータ
□ 入力者マスタのダミーデータ
```

---

### Phase 2: バックエンドAPI実装

#### Step 2- believers認証API
```
□ ログインAPI（POST /api/auth/login）
  - ID/PASS検証（2008/2008）
  - セッション管理
```

#### Step 2-2: マスタ系API
```
□ 製造依頼元API（GET/POST/PUT /api/departments）
□ 入力者API（GET/POST/PUT /api/employees）
□ 製品API（GET/POST/PUT /api/products）
```

#### Step 2-3: 実績系API
```
□ 実績一覧API（GET /api/achievements）
  - フィルター機能（日付、製造依頼元）
  - ページネーション
  
□ 実績登録API（POST /api/achievements）
  - 複数件一括登録対応

□ 実績更新API（PUT /api/achievements/:id）

□ 実績削除API（DELETE /api/achievements/:id）

□ 集計API（GET /api/summary）
  - 合計金額計算
```

#### Step 2-4: バリデーションミドルウェア
```
□ 入力値検証
□ エラーハンドリング
```

---

### Phase 3: フロントエンド基盤構築

#### Step 3-1: Next.jsプロジェクトの作成
```
□ frontend/ディレクトリを作成
□ npx create-next-app@latest
□ TypeScript設定
□ Tailwind CSS設定
```

#### Step 3-2: APIクライアント実装
```
□ src/api/api.ts を作成
□ axios または fetch でAPI呼び出し
□ TypeScript型定義
```

#### Step 3-3: 共通レイアウト実装
```
□ Headerコンポーネント
□ Sidebarコンポーネント
□ 認証管理（セッション）
```

---

### Phase 4: フロントエンド画面実装

#### Step 4-1: ログイン画面
```
□ src/app/login/page.tsx
□ ID/PASS入力フォーム
□ バリデーション
```

#### Step 4-2: ホーム画面
```
□ src/app/home/page.tsx
□ ナビゲーション（実績一覧、実績入力、実績修正）
```

#### Step 4-3: 実績入力画面
```
□ src/app/input/page.tsx
□ ドロップダウン（入力者、製造依頼元、製品名）
□ 複数件入力機能
□ 確認画面（src/app/input/review/page.tsx）
```

#### Step 4-4: 実績修正画面
```
□ src/app/edit/page.tsx
□ 一覧表示
□ ページネーション
□ 編集機能
```

#### Step 4-5: 一覧画面（管理者用）
```
□ src/app/list/page.tsx
□ フィルター（日付、製造依頼元）
□ 集計表示
□ 単価表示（管理者のみ）
```

---

### Phase 5: 統合テスト

#### Step 5-1: ローカル環境での動作確認
```
□ バックエンド起動（localhost:3000）
□ フロントエンド起動（localhost:4000）
□ 各画面の動作確認
□ エラーハンドリング確認
```

#### Step 5-2: UI/UX調整
```
□ 初心者でも使いやすいUI
□ 大きめのボタン
□ わかりやすい文言
```

---

### Phase 6: VPS環境構築 & デプロイ（社長承認後）

#### Step 6-1: VPS契約
```
□ エックスサーバーVPS契約（2GBプラン）
□ Ubuntu 24.04 LTS選択
```

#### Step 6-2 サーバー環境構築
```
□ SSH接続
□ PostgreSQLインストール
□ Node.jsインストール
□ PM2インストール（自動起動用）
```

#### Step 6-3: アプリケーションデプロイ
```
□ Gitでコードをクローン
□ データベース作成・マイグレーション
□ バックエンド起動
□ フロントエンドビルド
```

#### Step 6-4: セキュリティ・運用設定
```
□ Firewall設定
□ SSL証明書設定（HTTPS化）
□ 自動バックアップ設定
```

#### Step 6-5: 本番環境での動作確認
```
□ 社内のPCからアクセス確認
□ 複数ユーザーでの同時アクセステスト
```
